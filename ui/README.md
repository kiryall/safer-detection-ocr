# UI структура SEFER Vision

## Описание

Модуль `ui` содержит Streamlit приложение для обработки фотографий табличек с помощью SEFER Vision пайплайна.

## Структура папок

```
ui/
├── __init__.py                 # Инициализация пакета
├── app.py                      # Главная точка входа Streamlit приложения
├── streamlit_app.py            # Альтернативная версия (устаревшая)
├── pipeline_wrapper.py         # Обертка для запуска пайплайна
├── components/                 # UI компоненты
│   ├── __init__.py
│   ├── sidebar.py              # Боковая панель настроек
│   ├── progress.py             # Компонент прогресса
│   ├── metrics.py              # Метрики и статистика
│   └── results_display.py      # Отображение результатов
└── utils/                      # Утилиты
    ├── __init__.py
    ├── config.py               # Конфигурация приложения
    └── helpers.py              # Вспомогательные функции
```

## Компоненты

### `app.py`
Главный файл приложения. Содержит:
- Конфигурацию страницы Streamlit
- Инициализацию состояния
- Управление потоком обработки
- Отображение результатов

**Запуск:**
```bash
streamlit run ui/app.py
```

### `components/sidebar.py`
Боковая панель с настройками пайплайна:
- Префикс имён файлов
- Пути входных/выходных папок
- Лимит изображений
- Пороги уверенности (YOLO, OCR, CONF)
- Дополнительные опции (визуализация, дебаг, и др.)

**Экспортирует:**
- `PipelineConfig` — dataclass с конфигурацией
- `render_sidebar()` — функция для отрисовки панели

### `components/progress.py`
Управление и отображение прогресса обработки:
- Инициализация состояния через `session_state`
- Обновление прогресса в реальном времени
- Отображение прогресс-бара и текущего файла
- Спиннер загрузки

**Экспортирует:**
- `init_progress_state()` — инициализация
- `update_progress()` — обновление
- `render_progress_bar()` — отрисовка
- `reset_progress()` — сброс

### `components/metrics.py`
Отображение метрик и статистики:
- Количество успешных/сомнительных/ошибочных результатов
- Время обработки

**Экспортирует:**
- `render_status_metrics()` — 4 метрики в колонках
- `render_processing_time()` — время выполнения

### `components/results_display.py`
Отображение результатов обработки:
- Лучший результат
- Худший результат
- Таблица всех результатов
- Скачивание CSV отчёта

**Экспортирует:**
- `render_best_result()` — лучший результат
- `render_worst_result()` — худший результат
- `render_results_table()` — таблица со скачиванием

### `pipeline_wrapper.py`
Обертка для запуска пайплайна с отслеживанием прогресса:
- Класс `PipelineRunner` для управления обработкой
- Callback для обновления UI в реальном времени
- Обработка ошибок

**Экспортирует:**
- `PipelineRunner` — класс для запуска пайплайна

### `utils/config.py`
Конфигурация приложения:
- Переиспользует все константы из `configs/config.py` (без дублирования)
- Добавляет UI-специфичные константы (APP_TITLE, PAGE_LAYOUT, SIDEBAR_STATE)
- Вычисляет пути к отчётам

### `utils/helpers.py`
Вспомогательные функции:
- `add_project_to_path()` — добавление корня в PATH
- `validate_input_directory()` — валидация входной папки
- `load_results_csv()` — загрузка результатов
- `format_confidence()` — форматирование уверенности
- `get_status_emoji()` — эмодзи для статусов

## Использование

### Локально (для разработки)

```bash
# Установка зависимостей (если не установлены)
pip install streamlit pandas

# Запуск приложения
cd d:\data_science\safer
streamlit run ui/app.py
```

Приложение откроется по адресу: `http://localhost:8501`

### Конфигурация

При запуске `app.py` используются значения из `configs/config.py` (главная конфигурация проекта):
- `CONF_THRESHOLD` — порог для статуса 'success'
- `OCR_ACCEPT_THRESHOLD` — минимальный confidence OCR
- `YOLO_CONF_THRESHOLD` — порог уверенности YOLO
- `CONFIDANCE_LEVEL_THRESHOLD` — общий порог (det_conf * text_conf)

**Примечание:** UI не дублирует конфигурацию, а использует единую конфигурацию проекта.

Все параметры можно изменить в боковой панели приложения.

## Интеграция с пайплайном

Приложение использует функцию `run_pipeline()` из `src.pipeline.main_pipeline` с поддержкой:
- Callback функции для отслеживания прогресса
- Параметра `show_progress=False` для отключения tqdm
- Все параметры пайплайна передаются через UI

## Session State

Приложение использует Streamlit `session_state` для:
- Сохранения состояния прогресса между перезагрузками
- Кэширования результатов обработки
- Управления флагом `processing` (идет ли обработка)

## Примеры использования

### Базовая обработка
1. Запустить приложение
2. Оставить настройки по умолчанию
3. Нажать "🚀 Запустить обработку"
4. Дождаться завершения

### Кастомные параметры
1. В боковой панели изменить:
   - Папку с фото: `data/custom`
   - Префикс: `CUSTOM`
   - Пороги уверенности
2. Включить опции (визуализация, дебаг и др.)
3. Запустить обработку

### Скачивание результатов
После завершения обработки в таблице результатов можно:
- Просмотреть статистику
- Посмотреть примеры лучшего/худшего результатов
- Скачать полный CSV отчёт через кнопку "📥 Скачать полный CSV-отчёт"

## Зависимости

- `streamlit` — фреймворк UI
- `pandas` — работа с данными
- `pillow` (опционально) — обработка изображений для отображения

## Заметки

- При первом запуске приложение загружает модели YOLO и OCR, что может занять время
- Состояние приложения сохраняется в `session_state` и сбрасывается при перезагрузке страницы
- Для отключения кэша можно использовать флаг `--logger.level=debug`
