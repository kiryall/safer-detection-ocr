# SEFER Vision UI — Руководство по запуску

## 🚀 Быстрый старт

### 1. Убедитесь, что все зависимости установлены

```bash
pip install streamlit pandas pillow
```

### 2. Запустите приложение

```bash
# Из корня проекта
streamlit run ui/app.py
```

или

```bash
# С указанием полного пути
cd d:\data_science\safer
streamlit run ui/app.py
```

### 3. Откройте браузер

Приложение автоматически откроет браузер на `http://localhost:8501`

## 🎯 Как использовать приложение

### Шаг 1: Настройка параметров (боковая панель)

1. **Префикс имён файлов** — задайте префикс для переименованных файлов (по умолчанию: SEFER)
2. **Папка с фото** — выберите папку с входными изображениями (по умолчанию: `data/raw`)
3. **Выходная папка** — папка для сохранения результатов (по умолчанию: `output`)
4. **Лимит изображений** — максимальное количество (0 = обработать все)

### Шаг 2: Установка порогов уверенности

- **YOLO Detection Confidence** — порог детекции табличек (0.1–0.95)
- **CONF_THRESHOLD** — порог для статуса "успешно" (0.1–0.95)
- **OCR Accept Confidence** — минимальная уверенность OCR (0.0–1.0)

### Шаг 3: Дополнительные опции

- ☑️ **Визуализация OCR** — сохранять визуализации распознавания
- ☑️ **Сохранять кропы для дебага** — сохранять вырезанные таблички
- ☑️ **Не перемешивать порядок файлов** — обработать в исходном порядке
- ☑️ **Очистить выходную папку** — удалить старые результаты перед запуском

### Шаг 4: Запуск обработки

Нажмите **🚀 Запустить обработку**

- Приложение загрузит модели (первый запуск может занять время)
- Вы увидите прогресс-бар с текущим файлом
- После завершения будут показаны результаты

### Шаг 5: Просмотр результатов

Приложение отобразит:

1. **Время обработки** ✅
2. **Метрики** (успешные, сомнительные, ошибки, всего)
3. **Примеры** (лучший и худший результаты)
4. **Полная таблица** со всеми результатами
5. **Кнопка скачивания** CSV отчёта

## 🔧 Примеры команд (для PowerShell)

### Базовый запуск
```powershell
streamlit run ui/app.py
```

### С указанием порта
```powershell
streamlit run ui/app.py -- --server.port 8502
```

### В режиме без браузера
```powershell
streamlit run ui/app.py -- --logger.level=debug
```

### Очистить кэш и запустить
```powershell
Remove-Item -Path "$env:TEMP\streamlit" -Recurse -Force -ErrorAction SilentlyContinue
streamlit run ui/app.py
```

## 📊 Структура результатов

После обработки в папке `output/` будут созданы:

```
output/
├── success/              # ✅ Успешно распознанные
├── low_confidence/       # ⚠️ Сомнительные (низкая уверенность)
├── very_low/            # ❌ Не распознано (очень низкая)
├── visualizations/      # 📊 Визуализации OCR (если включено)
├── crops/
│   └── success/         # 🖼️ Кропы табличек
├── sefer_report.csv     # 📋 CSV отчёт
└── sefer_summary.txt    # 📝 Текстовый отчёт
```

## 🐛 Troubleshooting

### Проблема: "Модуль не найден"
**Решение:** Убедитесь, что запускаете из корня проекта
```bash
cd d:\data_science\safer
streamlit run ui/app.py
```

### Проблема: "Папка с фото не найдена"
**Решение:** Проверьте путь в боковой панели, используйте относительный путь от корня проекта

### Проблема: Долгая загрузка при первом запуске
**Решение:** Это нормально, первый запуск загружает модели YOLO и PaddleOCR. Последующие запуски будут быстрее.

### Проблема: "Port already in use"
**Решение:** Используйте другой порт:
```bash
streamlit run ui/app.py -- --server.port 8502
```

## 💡 Советы

1. **Первый запуск** может занять 2-5 минут (загрузка моделей)
2. **Результаты кэшируются** между перезагрузками приложения
3. **Для больших наборов** используйте лимит файлов для тестирования
4. **Визуализация OCR** полезна для дебага, но замедляет обработку
5. **CSV отчёт** содержит все детали для дальнейшего анализа
